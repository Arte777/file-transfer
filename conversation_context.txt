1. ИСТОРИЯ ЗАПРОСОВ И ХРОНОЛОГИЯ ДИАЛОГА
--------------------------------------------------------------------------------
* Запрос 1: Инициализация разработки новой версии 7.2.2. Замена Runtime Broker на актуальный билд в установщиках, обновление структуры папок. Требование добавить уникальные аватарки/иконки для каждого приложения (как на этапе установки, так и в системе после нее). Сборка релизов RAH NON PRO и RAH PRO (standalone).
* Запрос 2-7: Жалобы на бесконечную загрузку, зависание и отсутствие старта приложений после установки. Диагностика проблем с отсутствием .NET зависимостей на целевых ПК.
* Запрос 8: Требование собрать полностью автономные установщики со встроенным рантаймом .NET, проследить за отображением правильной версии (вместо 0.0.0.0) и иконок.
* Запрос 9: Выгрузка актуальных билдов установщиков на сервер/сайт (docs/downloads/).
* Запрос 10: Редизайн кнопки группового обновления на веб-панели (отображение количества устаревших ПК, требующих обновления). Активация кнопки для оператора Dildman.
* Запрос 11: Уточнение по поводу того, что старые клиенты v7.0.x не имеют функций автообновления и не должны учитываться счетчиком. Исправление логики счетчика, чтобы он считал только клиентов начиная с поддерживающей версии (>= 7.2.2).
* Запрос 12: Исправление критических вылетов (крашей) установленных приложений на старте (ошибка XamlParseException, связанная со встраиванием ресурса иконки app.ico).

2. ДЕТАЛЬНЫЙ СПИСОК ВЫПОЛНЕННЫХ ТЕХНИЧЕСКИХ РАБОТ
--------------------------------------------------------------------------------

А. WPF КЛИЕНТЫ (client-wpf и standalone-shonll):
* Исправление краша XamlParseException:
  - Проблема: Атрибут Icon="app.ico" в файлах разметки MainWindow.xaml приводил к падению приложения на компьютерах без исходного кода, так как компилятор .NET не вшивал иконку внутрь исполняемого файла из-за типа сборки "None".
  - Решение: Тип сборки для файла app.ico изменен на "Resource" в файлах проектов (.csproj). Из MainWindow.xaml удалены относительные атрибуты Icon. В конструктор MainWindow.xaml.cs добавлена безопасная программная загрузка ресурса иконки через Pack URI ("pack://application:,,,/app.ico").
* Полная автономность сборки (Self-Contained):
  - Публикация проектов переведена на режим полной автономности со встроенной средой выполнения .NET 8.
  - Команда сборки: `dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -p:EnableCompressionInSingleFile=true`
  - Вес исполняемых файлов вырос с ~3 МБ до ~65 МБ, обеспечивая запуск на любой системе без необходимости ручной установки .NET Runtime.

Б. ИНСТАЛЛЯТОРЫ INNO SETUP (setup/shonll_client.iss и setup/shonll_standalone.iss):
* Исправление версии установщика (0.0.0.0 -> 7.2.2):
  - В секцию [Setup] обоих скриптов добавлены свойства версии:
    VersionInfoVersion=7.2.2
    VersionInfoTextVersion=7.2.2
    VersionInfoCompany=Убежище
    VersionInfoDescription=RAH NON PRO / RAH PRO Setup
* Настройка запуска от имени администратора:
  - Установлен параметр PrivilegesRequired=admin, запрашивающий повышение прав (UAC) при старте инсталлятора.
* Обновление дистрибутива:
  - Установщики перекомпилированы с помощью ISCC.exe, в них упакованы новые self-contained версии клиентов. Готовые файлы RAH_Non_Pro_setup.exe (64.3 MB) и RAH_PRO_setup.exe (63.4 MB) скопированы в веб-директорию загрузок docs/downloads/.

В. ВЕБ-ПАНЕЛЬ И СЕРВЕРНАЯ ЛОГИКА (docs/ и server/):
* Редизайн кнопки "Обновить всё" на дашборде:
  - Кнопка оформлена в виде статус-карточки, показывающей количество устаревших ПК в реальном времени (например, "26 ПК").
  - Кнопка активирована и доступна для использования как для Shonll, так и для dildman.
* Фильтрация несовместимых версий:
  - В docs/assets/dashboard.js добавлена функция `supportsUpdate(ver)` и изменена `isOutdated(ver)`. Теперь клиенты версий 7.0.x и 7.2.1 не учитываются счетчиком обновлений, так как физически не имеют механизма фонового скачивания и автозапуска. Счетчик считает только клиентов >= 7.2.2, имеющих версию ниже актуальной.
  - На бэкенде в server.js изменен роут `/request-update-all`. База данных MongoDB обновляет флаг запроса обновления только у тех компьютеров, чья версия поддерживает автообновление и не равна текущей актуальной версии.
* Деплой проекта:
  - Все изменения закоммичены в git-репозиторий и успешно отправлены на хостинг Railway. Контейнер бэкенда пересобран и запущен.
